# Security Policy

## Supported Versions

Currently, only the latest `main` branch version of DialerJazz is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| main (latest) | :white_check_mark: |
| older commits | :x:                |

## Reporting a Vulnerability

Please **DO NOT** report security vulnerabilities via public GitHub issues.

Instead, please email the maintainer directly at **moazzam07@proton.me** (or contact via GitHub profile). We take security very seriously and will triage and fix vulnerabilities as quickly as possible.

When reporting a vulnerability, please provide:
* The type of issue (e.g., SQL injection, cross-site scripting, auth bypass).
* Full paths of source file(s) related to the manifestation of the issue.
* The location of the affected source code (tag/branch/commit or direct URL).
* Any special configuration required to reproduce the issue.
* Step-by-step instructions to reproduce the issue.
* Proof-of-concept or exploit code (if possible).
* Impact of the issue, including how an attacker might exploit the issue.

We will endeavor to respond to your report within 48 hours.
